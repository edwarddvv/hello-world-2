**git fetch** is used to get remote changes on to your computer, but it leaves your local copy unchanged.  In that way, you can view the remote changes and decide whether you want to incorporate them into your code.  If you do, you run **git merge**.  Then your local copy will be the same as the remote one.  The changes could have been made by a colleague and you have to decide whether you want to incorporate them into your branch at this time.  It could be that the change is not relevant to the feature that you are working on in your branch and you would rather finish implementing the feature before incorporating the colleague's change.

**git pull** does these two steps in one go i.e. it collects the changes from the remote repository and applies them to your local copy.  This is useful if you are just using the code and you are told that a bug has been discovered in it and you should apply the bug fix.
